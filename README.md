README: Book My Stay – Hotel Booking Management SystemProject OverviewBook My Stay is an incremental Java-based project designed to illustrate the practical application of Core Java and fundamental data structures in a real-world software engineering context. Instead of focusing on a user interface, this project emphasizes backend logic, system behavior, and data integrity.The system addresses common challenges in scalable software, such as:Fairness: Ensuring first-come, first-served request handling.Consistency: Preventing double-booking and maintaining accurate inventory.Concurrency: Handling multiple simultaneous users safely.Persistence: Saving and recovering system state across restarts.Project Evolution (Use Cases)The system is built through a series of progressive use cases, each introducing a critical software engineering concept:Use CaseTitleKey ConceptData StructureUC4Room SearchRead-Only AccessMap, EncapsulationUC5Booking RequestFIFO FairnessQueue<Reservation>UC6Room AllocationUniqueness EnforcementSet<String>, HashMapUC7Add-On ServicesBusiness ExtensibilityMap<String, List<Service>>UC8History & ReportingOperational VisibilityList<Reservation>UC10CancellationState Reversal (Rollback)Stack<String>UC11Thread SafetyRace ConditionsSynchronized, ConcurrentHashMapUC12Data PersistenceDurability & RecoverySerialization, File I/OCore PrinciplesSeparation of Concerns: Search logic, booking intent, and actual room allocation are isolated to prevent unintended side effects.Defensive Programming: Validation checks ensure that only valid room types and available units are processed.Atomic Operations: Allocation and inventory updates occur together to ensure the system never enters an inconsistent state.LIFO Rollback: Using a Stack for cancellations ensures the most recently assigned room is the first to return to the pool.Technical RequirementsLanguage: Java 8 or higher.Tools: Standard Java Compiler (javac) and Runtime (java).External Libraries: None (Uses standard java.util.*, java.util.concurrent.*, and java.io.*).How to Run the ProjectEach use case is implemented as a standalone executable file to demonstrate specific functionality. To run the final iteration (Data Persistence):Compile the code:Bashjavac UseCase12DataPersistenceRecovery.java
Run the application:Bashjava UseCase12DataPersistenceRecovery
Verify Persistence:Run the program once to create a booking, then exit. Run it a second time to see the system recover the previous bookings from the hotel_data.ser file.Key Benefits for LearnersUnderstand "Why" over "How": Learns why a Queue is better than a List for request handling, or why Serialization is vital for business continuity.Architecture Mindset: Moves beyond simple "Hello World" apps to modular service-based design.Real-World Simulation: Models actual business scenarios like peak-demand booking spikes and cancellation rollbacks.
